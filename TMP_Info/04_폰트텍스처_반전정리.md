# 폰트 텍스처/글리프 "반대로 보이는" 케이스 정리

생성시각: 2026-02-24

## 1) 실제 글리프 Y 좌표를 뒤집는 케이스 (데이터 변경)

- 대상: 신형 TMP(`m_GlyphTable/m_CharacterTable`)를 구형 TMP(`m_glyphInfoList`)로 내릴 때
- 판별 함수: `detect_new_glyph_y_flip(...)`
- 근거 코드:
  - `detect_new_glyph_y_flip` 정의: `unity_font_replacer_core.py:1184`
  - 점수 비교식(`yy = atlas_h - y - h if flip_y else y`): `unity_font_replacer_core.py:1241`
  - flip 최종 조건(비영역 개수/평균 밝기 비교): `unity_font_replacer_core.py:1268`
  - 실제 반영식(`rect_y = atlas_height - rect_y - rect_h`): `unity_font_replacer_core.py:1295`
  - 적용 지점(`flip_y=flip_new_glyph_y`): `unity_font_replacer_core.py:2898`

요약:
- 이 케이스는 **프리뷰용 보정이 아니라 실제 저장되는 구형 glyph 좌표(y)** 를 바꿉니다.
- 즉, 구형 TMP 타겟에서는 글리프가 위아래 뒤집혀 보이는 문제를 이 경로에서 교정합니다.

## 2) 프리뷰에서만 상하 반전하는 케이스 (데이터 미변경)

- 대상: 대상 게임 Atlas를 "검증용 preview"로 만들 때
- 근거 코드:
  - `ImageOps.flip(preview_image)`: `unity_font_replacer_core.py:2425`
  - 주석: linear(non-swizzled) 텍스처는 Unity 좌표계(Y=0 하단)라서 프리뷰용 상하 반전 보정

요약:
- 이 케이스는 **미리보기 이미지에서만** flip 합니다.
- 실제 교체 저장되는 atlas 바이트 자체를 뒤집는 동작이 아닙니다.

## 3) PS5 swizzle/unswizzle로 인해 "회전/방향"이 달라 보이는 케이스

- 상수: `PS5_SWIZZLE_ROTATE = 90` (`unity_font_replacer_core.py:30`)
- 변환 함수:
  - swizzle 적용 전 역회전: `apply_ps5_swizzle_to_image` (`unity_font_replacer_core.py:704`, `:710`)
  - unswizzle 후 정회전: `apply_ps5_unswizzle_to_image` (`unity_font_replacer_core.py:732`, `:748`)
- 상태 판별:
  - `detect_texture_object_ps5_swizzle_detail(...)` (`unity_font_replacer_core.py:772`)
  - 메타 힌트 우선 판정(stream/readable/format 기반): `unity_font_replacer_core.py:806` 부근
- 최종 적용 지점:
  - 타겟 swizzle 상태 추정 후 `atlas_for_write`를 swizzle/unswizzle 변환: `unity_font_replacer_core.py:2964`~`3018`

요약:
- 여기서는 단순 Y-flip이 아니라 **swizzle 상태 + 90도 회전 규칙**이 개입합니다.
- 판정이 틀리면 결과가 "뒤집힘"처럼 보일 수 있습니다.

## 4) 왜 "반대로" 보일 수 있는가 (원인 분류)

- A. 구형 TMP 변환 시 glyph 좌표계 차이(실제 y값 변환 필요)
- B. 프리뷰에서만 보정된 이미지를 실제 결과로 오해
- C. PS5 swizzle 상태 판정/힌트(`swizzle=yes`, `process_swizzle=True`)와 타겟 실제 상태 불일치

## 5) 빠른 확인 체크리스트

- 로그에 `구형 TMP 좌표계 보정(Y-flip) 적용`이 찍혔는지 확인
- 로그에 `PS5 swizzle 감지: 대상 Atlas가 swizzled/linear` 판정이 무엇으로 나왔는지 확인
- preview PNG와 실제 write 대상 atlas 상태를 구분해서 비교
